Account and ALC Registration

Account and ALC Registration is the process by which a representative of a customer organization or IPAC processing activity requests that the data associated with an Account Code or ALC be made available for viewing in VCSS.  In the case of an ALC, this is the transactions where the ALC is used as the "designated agent" for an IPAC transaction.

 

The individual making the request must be the individual the organization to which the Account Code or ALC was issued wishes to be the "account manager" for the code.  The "account manager" will be responsible for approving or disapproving requests from other users that their user id's be enabled to see the data associated with an Account Code or ALC for which they are the "account manager".

 

To begin the registration process, navigate to http://vcss.gsa.gov and click the "Register your company/agency in VCSS" link.

 

 

On the next screen, select the "Customer" radio button and click "Submit".

 

 

On the next screen, enter a complete Account Code, ALC, or BPN.  An error message will be returned if there is no such Account Code, ALC, or BPN in the system, or if some other user has already registered the code.

 

 

 

 

On the next screen, fill in the required information and click "Review Registration" .

 

 

 

On the next screen, review the information entered to ensure there are no typographical errors, and "submit" the request to the GSA.

 

The GSA will review the request, and send an email to the "account manager".  If the "account manager" does not already have a user id, the GSA will provide a user id and password via separate emails.
